Features

More than a translation plugin.

A complete multilingual infrastructure — from content and SEO to export/import, background processing and developer APIs.

Smart Background Queue

Action Scheduler (v3.9.3 bundled) or WP Cron fallback. Each item runs as a separate AS action — retry built in. Stale processing recovery: stuck items auto-reset after 5 minutes.

ASYNC · RECOVERABLE · AS v3.9.3

Round-Robin API Keys

Add multiple API keys per provider — the plugin rotates through them automatically. Rate limit hit? Retry-After header respected. Exponential backoff: 2s → 4s → 8s. All 6 providers covered.

RATE LIMIT SAFE · ALL PROVIDERS

Gutenberg Block Optimizer

Only innerContent is translated — block structure and HTML comments preserved intact. 2× fewer API calls = 50% cost reduction on block-heavy content.

COST OPTIMIZED · 50% LESS TOKENS

XLSX / CSV Export & Import

Posts, terms, menus, ACF fields, Polylang strings and theme PO files — all in one XLSX. Send to translators, edit in Excel, import back. Dry-run shows diff before saving. Skip-same skips unchanged.

BATCH IMPORT · DRY-RUN · DIFF LOG

SEO Meta Translation

Yoast SEO + Rank Math: meta description, SEO title, Open Graph tags, image alt text. Translate existing or generate fresh with AI. Per-field control via settings.

YOAST · RANKMATH · OG TAGS

Translation Lock

Per-post lock switch in the meta box. Locked posts are skipped in all bulk operations — Translate All, background queue, auto-translate on save. Zero risk of overwriting manual translations.

PER-POST · META BOX TOGGLE

ACF Field Translation

All text, textarea and wysiwyg ACF fields translated automatically alongside the post. Same API request — no extra calls. Field-level source tracking in translation logs.

ACF NATIVE · SAME REQUEST

PO / MO File Translation

Translate theme and plugin .po files with AI. Native PHP MO compiler — no msgfmt binary needed. Plural forms auto-detected per language (tr/ja: 1, en/de: 2, ru/pl: 3). Stage 2 string editor included.

NO BINARY · PLURAL FORMS · MO COMPILE

Smart Menu Translation

Menu items linked to posts, pages or terms reuse existing Polylang translations — zero AI calls. WC endpoint URLs auto-updated via pll_translate_url(). Only truly custom items hit the API.

SMART RESOLVE · ZERO WASTE

How it Works

Up and running in 5 minutes.

1

Activate the Plugin

Polylang, WPML or qTranslate-XT auto-detected on boot. No configuration needed for ML plugin integration.

2

Settings → Translator

Pick your AI provider. Paste one or more API keys. Multiple keys enable round-robin load balancing automatically.

3

Check Untranslated

Go to Posts or Terms page. Select languages and post types. Hit Check Untranslated — the list populates instantly with cost estimate.

4

Translate All or Queue

Inline mode translates in the browser. Background queue mode uses Action Scheduler — leave the page, it keeps running.

5

Export / Import for Review

Export everything to XLSX, share with translators, import reviewed file back. Diff mode shows exactly what changed.

functions.php — developer API
// Translate a single post programmatically $integration = sat_plugin() ->getContainer() ->get('integration'); $translatedId = $integration ->translatePost(1661, 'de'); // Add item to background queue QueueManager::addItem( 'post', 1661, 'tr' ); // Batch add to queue QueueManager::addBatch( 'post', [1661, 1662, 1663], ['tr', 'de', 'fr'] ); // Check queue status $status = $queue->getStatus('post'); // → pending, processing, done, error, percent
